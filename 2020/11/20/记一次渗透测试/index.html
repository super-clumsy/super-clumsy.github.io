记一次渗透测试

|字数总计:3k|阅读时长:9分钟|阅读量:

前言

这次HW，了解到了真实的渗透测试流程，深刻了解到了除了ctf，一名合格的web狗更要学会渗透测试

因为是第一次，所以学到了大部分的工具，在这里整理一下，渗透测试遇到的一些基本知识点,以及一次打到一半的复现

实战开始

任务

此次任务，渣渣我第一次接触红队，才知道他们只提供一个名单，也叫靶标，比如

xxx公司 xxx企业
xxx机构 xxx单位

没错，就这么水。。。只提供一个名字，然后让我们去找相关目标的信息，开始网上搜集信息，找到他们的网站，看一看有没用什么缺口，漏洞，然后控制后台，占领服务器等等，都会获得加分。

好，废话不多说，开始

流程

流程一共很多步骤，其实kali，就基本全部列出了

但是主办方的电脑不让用虚拟机，于是基本下都是在windows上操作的。

其中最莫过于重要的可能是信息搜集，好歹知道打谁，才有机会打.

1.信息搜集

  • 通过百度(感觉百度是最废的)，谷歌，fofa，钟馗之眼，潮汐指纹等搜索对应的公司名称，得到一个主站以及对应的IP

    这都是网页版本的查询，具体查询都有对应的语法

    如: intitle: intext: site= ip=搜索敏感词汇

    确定了大体目标以后，我们要保证他是真实 IP,大部分都会存在CDN，CDN是为了缓解服务器压力而出现的，他会阻碍我们的基础进程，所以要绕过(CDN大家可以自行百度)

  • 通过nslookup 或者 站长之家的超级ping 来确定是否有CDN

    超级ping

    nslookup 则是cmd下的一个命令

    如果查询出的ip数量大于一个的话，则说明该ip地址不是真实的服务器地址。也就是有CDN

  • 如果判断出有CDN，则需要绕过CDN，然后用相应的工具绕过CDN，确保自己打的是真实的ip

    1. 因为CDN不是很便宜，所以一般只给流量大的站点做CDN，很多小站子站点，又跟主站在同一台服务器或者同一个C段内，此时就可以通过查询子域名对应的 IP 来辅助查找网站的真实IP

      于是我们需要子域名挖掘机

    2. 可以通过网站查找CDNCDN查询，不停的绕，不停的判断，直到ping不出多个IP为止

    3. 通过国外网站进行访问ping，因为CDN一般只对国内用户进行加速

  • 然后用搜集相应的IP相关的信息,站长之家的whois，以及工具网站,进可能的对目标IP，获取更多的信息

  • 还可以在贴吧，论坛等等地方，查询与该公司有关的任何信息。比如，公司的邮箱格式，公司的员工姓名，以及与该公司有关的任何信息。(指不定哪就进入公司的企业邮箱去了)

  • 用工具判断服务所使用的web容器，所使用的CMS，这样可以方便查找相关漏洞

    可以使用网站指纹识别工具:云悉指纹识别 在线cms识别

  1. 服务器类型: ping 一下，如果TTL值是两位 如64，就是linux系统，而三位数,如128,z则是windows系统
  2. nmap 进行扫描 nmap -A ,可以扫描出网站服务器对应的版本

  1. 根据插件 wapplayzer 判断 脚本类型(此步决定了以什么脚本编写木马并上传)

      • ASP 和 ASPX：ACCESS、SQL Server

      • PHP：MySQL、PostgreSQL

      • JSP：Oracle、MySQL

        一般都为这几种

  • GOBY工具扫描开放的端口，如 135 、137 、138 、139 、445，这几个端口经常爆发漏洞

  • 然后 扫描网站敏感目录和文件

    1. 后台目录：弱口令，万能密码，爆破
    2. 安装包：获取数据库信息，甚至是网站源码
    3. 上传目录：截断、上传图片马等
    4. mysql管理接口：弱口令、爆破，万能密码，然后脱裤，甚至是拿到shell
    5. 安装页面 ：可以二次安装进而绕过
    6. phpinfo：会把你配置的各种信息暴露出来

  • 最后的最后，就是C段，有些网站可能不是那么容易入侵。那么，可以查看该网站所在的服务器上是否还有其他网站。如果有其他网站的话，可以先拿下其他网站的webshell，然后再提权拿到服务器的权限，最后就自然可以拿下该网站了！

    c段扫描

    还有网上给的一些扫描工具

    至此，信息搜集完毕

2.漏洞探测

  • SQL注入 (基本能过滤的都过滤了，反正我是没见到过)
  • XSS跨站脚本
  • CSRF跨站请求伪造
  • XXE漏洞
  • SSRF服务端请求伪造漏洞
  • 文件包含漏洞
  • 文件上传漏洞
  • 文件解析漏洞
  • 远程代码执行漏洞
  • CORS跨域资源共享漏洞
  • 越权访问漏洞
  • 目录浏览漏洞和任意文件读取/下载漏洞，
  • struts2漏洞，
  • JAVA反序列化漏洞

这些漏洞需要日常积累，当然，如果工具扫描出来了，一般都是CVE编号的漏洞，可以配合msf进行攻击，尝试控制该计算机，普通的burp等等工具自然也是一把好手

小白，我见到的只有文件上传漏洞(哥斯拉了解一下)

3.内网转发

对于我们得到漏洞，我们通过上传大马，来控制此计算机，此时，我们需要打进内网，才能进一步扩大我们的权限

域渗透，一步步扩大自己的权限

内网转发先鸽一下，大家参考内网渗透

4. 内网横向渗透

简单来说，就是从一台计算机，跳到另一台计算机上

(烂土豆提权了解一下)

对于windows主机，我们应该多去翻翻目录，或许能有很多意想不到的结果。很多人习惯把账号密码等容易忘的东西存放在备忘录中，或者是桌面上。我们还可以查找数据库的连接文件，查看数据库的连接账号密码等敏感信息。当我们获得了windows主机的账号密码，或者是自己创建了新用户后，我们为了不被网站管理员发现和不破坏服务器。我们尽量不要使用远程桌面。因为使用远程桌面动静比较大，如果此时服务器管理员也在登录，而你此时通过远程桌面登录的话，会将管理员挤掉，而你也将很快的被管理员踢掉。对于实在是要远程桌面登录的情况的话，我们尽量不要新建一个用户进行登录。我们可以激活 guest 用户，然后将其加入 administrators 组里面，用 guest 用户身份登录。在RDP远程登录后，我们可以查看其他用户桌面上和其他目录有哪些软件，我们要找的目标有以下。

  • FTP相关的软件
  • 数据库相关的软件
  • 打开浏览器，查看历史记录，查看某些网站是否保存有用户密码。利用工具查看浏览器保存的密码

从该主机上找到的账号密码，我们可以做一个字典，在对内网其他机器进行爆破的时候，很有可能是同密码。

5.权限维持

在拿到目标主机的权限后，很有可能当时我们并不能获取到想要的东西，需要进行长期的潜伏，特别是在内网渗透中，需要进行长期的信息收集。这时，权限维持就很重要了。我们需要维持住获得的现有权限。

  1. Web后门

    • 隐藏后门文件(将文件设置为隐藏)
    • 404页面隐藏后门，或者在其他自带文件中插入后门代码(日志文件包含了解一下)
    • 不死马，该脚本每5秒向服务器创建test.php，并写入一句话免杀木马。结合attrib命令隐藏文件更好地建立后门。

    注：以上几种后门方法均能被D盾等工具检测到

  2. linux系统

    1：SSH后门

    2：crontab定时任务

    3：SSH公钥

    4：创建SUID=0的用户

  1. windows 系统

1: 建立隐藏用户，在用户名后加 $

2：在开机启动目录下放置木马文件，只要目标机器重启，将回连我们的远控

3：MSF里的 persistence 模块，执行命令可以使目标机器每隔一定时间自动回连远控。但是容易被发现

4：在域环境下，想办法获得 krbtgt 用户的哈希，该用户的哈希可以进行票据传递攻击。而且一般该用户的密码不经常改变。

5：shift后门(粘滞键替换)

6：远程桌面会话劫持
