指纹识别

了解CMS与指纹识别

CMS：快速搭建网站的内容管理系统

Web应用框架：快速二次开发的web应用框架，例如，网站和小程序等。

指纹的特性

  • 唯一性
  • 方便性
  • 终身不变性

指纹自动识别的方式

  • 云悉指纹识别 : https://www.yunsee.cn/

    • Wappalyzer:https://www.wappalyzer.com, 这个火狐插件都有
    • w3Techs

  • 脚本方式：CMSeek：

    手动识别

    这个需要人工

    HTTP Header

      • 观察是否有X-Powered-By字段

      • 观察Server字段

        可以间接判断支持的语言，主要可以判断所采用的中间件和服务器版本信息

      • cookies 中的 sessions字段

    可以看到 session id 写的 phpSessid， 说明是用php写成的，如果是jsession，说明语言是用java写的。

    其他方式

    • 观察url后缀名文件的后缀
    • 通过报错页面判断中间件

    以百度为例，输入 index.jsp后，显示页面不存在，那么就表明，该页面不是使用java语言开发的，

    当输入index.php时，页面回显正常们没有产生报错，说明该页面时采用php语言开发的。

当输入肯定不存在的路径的时候，会出现报错回显页面

  • apcahe 简单朴素。

  • IIS 则采用 头部为灰色，中间为鲜红的404的报错界面，十分容易识别。

  • nginx 会直接返回 nginx的报错信息

绕过cdn，寻找真实IP的方法

基本思路是 尽可能避开互联网上有可能影响数据传输速度和稳定性的瓶颈和环节，使内容传输的更快，更稳定

改善网络加载时间，减少服务器负载，降低成本，提高网络安全行

CDN检测

  • 国内在线 CDN云观测 http://cdn.chinaz.com
  • 国外在线 CDNplanet https://www.cdnplanet.com
  • 脚本探测：xcdn https://github.com/3xp10it/xcdn

常见的寻找真实IP的方法：

找真实IP是个体力活，需要各种耐心和运气成分。

  • 超级ping：http://ping.chinaz.com/
  • 历史解析：
    • 微步在线：https://x.threatbook.cn/
    • 历史Whois：https://www.benmi.com/whoishistory/
    • 历史托管：https://toolbar.netcraft.com/site_report?url=
  • 内部邮箱源：对方发送邮件，查看邮件来源IP
  • 二级域名：寻找子域名，找到未绑定CDN站点。
  • 手机App：使用Brup抓取收集App数据包
  • 微信公众号：使用Brup抓取微信公众号数据包
  • 国外ping：
    • ping.ge：http://www.ping.pe
    • 海外测速：http://tool.chinaz.com/speedworld/
  • 网络空间搜索引擎：fofa：https://fofa.so
