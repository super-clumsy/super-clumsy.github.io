青藤云HW面试

  1. 我看你的简历上有参加过HW，你有没有取得什么成就

    答:我去年参加过一次市级红队，经过信息搜集以后，发现了它存在目录遍历漏洞，找到了登录的日志文件，里面有用户名的登录信息，使用python脚本，匹配密码是123456的用户名，成功登录，使用哥斯拉上传木马，控制了服务器。之后由于能力问题，没有内网渗透的经验，止步于此

  2. 你既然用过哥斯拉，你知道他或者冰蝎的流量特征吗，以及他的加密方式。

    答: 哥斯拉是使用的AES加密，低版本的话会有特征，特征是强特征，

    • 发送一段固定代码（payload），http响应为空

    • 发送一段固定代码（test），执行结果为固定内

    • 发送一段固定代码（getBacisInfo）。

      冰蝎也是强特征

      Content-Type: application/octet-stream属于强特征，UA头则是是弱特征

  3. 你知道文件上传漏洞是怎么造成的吗？他常见的函数有哪些？

    答: 服务器配置不当

    ​ 本地文件上传限制被绕过

    ​ 服务端过滤不严格被绕过

    ​ 文件路径截断

    ​ 文件解析漏洞导致文件执行

    ​ 编辑器上传漏洞

    ​ 常见的错误可能有php中.htacess文件配置失误，或者解析漏洞以及常见的%00截断等

    ​ 这个函数我不太多见，我记得它应该可以有白名单和黑名单两种，比较保险点的是白名单，黑名单总有绕过的方法

  1. 如果现在的你，遇到到了去年HW的情况，你接下来该怎么做呢？

    答:接下来需要进行提权，获取system或者root权限，可以先试着用msf进行提权，如果失败，可以使用systeminfo或者uname -a查看当前系统版本的内核情况，考虑相应的提权，如==缓冲区溢出提权，烂土豆提权，mysql的udf，mof提权，以及suid或者脏牛提权等==，之后用此靶机当跳板，继续搜集内网里面的信息，或者进行横向，纵向渗透，扩大控制范围。

其他hw面试题目

1.如果领导让你应急一台服务器的入侵你会怎么做

答：①观察进程信息和连接情况，以及最近的登录信息。开始排查可疑程序。此时，为保护服务器和业务，避免服务器被攻击者继续利用，应尽快歉意业务，立即下线机器；如果不能立即处理，封掉该服务器对网络的双向连接。②分析。IP及所处区域拓扑等：VLAN内服务器和应用情况，确定同一网络下面服务器之间的访问，是否可以相互登录，由此确定影响的范围，确人所有受到影响的网段和机器。③进行在线分析：**关键字：wget/curl， gcc， 或者隐藏文件， 敏感文件后缀（.c，.py，conf， .pl， .sh)**。

同时找出root权限的用户grep -v -E "^#" /etc/passwd | awk -F： '$3 == 0 { print $1}' ,

2.溯源攻击者的途径

答：根据攻击者操作留下的日志文件进行审计，找到攻击者留下的后门，放在虚拟机里netstat -a查看连接木马的ip,攻击者通常会用迷惑的手法，伪装成正常的文件，甚至是系统文件来躲避管理员的查杀。

6.CSRF漏洞原理

​ 当在浏览器上登录某网站后，cookie会保存登录的信息，这样在继续访问的时候不用每次都登录了，而CSRF就利用这个登陆态去发送恶意请求给后端。

7.SSRF的防御措施

  • 限制协议为HTTP、HTTPS

  • 不用限制302重定向

  • 设置URL白名单或者限制内网IP

8.永恒之蓝用到的

MS17-010

永恒之蓝 主要就是通过TCP端口（445）和139 来利用SMBv1和NBT中的远程代码执行漏洞，恶意代码会扫描开放445文件共享端口的windows 机器，无需用户任何操作，只要开机上网，不法分子就能在电脑和服务器里进行一系列的危险操作。如远程控制木马，获取最高权限等

11.对tomcat、weblogic、shiro原理、利用方式、漏洞复现

tomcat：该漏洞可以任意文件类型解析为jsp，从而达到任意命令执行的效果。但漏洞需要配合文件上传漏洞才可利用

weblogic：未经身份验证的远程攻击者可能通过构造特殊的 HTTP GET请求，利用该漏洞在受影响的 WebLogic Server 上执行任意代码。
通过非法字符绕过访问，然后通过Gadget调用命令执行

shiro：Shiro框架提供了记住密码的功能（RememberMe），用户登录成功后会生成经过加密并编码的cookie。在服务端对rememberMe的cookie值，先base64解码然后AES解密再反序列化，就导致了反序列化RCE漏洞。

命令=>序列化=>AES加密=>base64编码=>RememberMe Cookie值

在整个漏洞利用过程中，比较重要的是AES加密的密钥，如果没有修改默认的密钥那么就很容易就知道密钥了,Payload构造起来也是十分的简单。
