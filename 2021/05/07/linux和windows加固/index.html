Linux系统加固

  2. 修改密码密码，确保最小长度为8位

  5. 不响应ICMP请求

  6. 结束非法登录用户

windows加固

  1. 修改3389端口

    3389端口懂的都懂，是个隐患，我们采取改动3389端口来加固系统

    单击[开始]—[运行]，输入regedit，打开注册表后，单击进入以下路径：

    a.HKEY_LOCAL_MACHINE/SYSTEM/CurrentControlSet/Control/Terminal Server中fDenyTSConnections，将其值改为0（默认为1）; //启用Terminal server

    b.HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control \Terminal Server\Wds\rdpwd\Tds\tcp，在此有一个portnumber，将其更改为其他端口号;

    c.HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp，这下面也有一个portnumber将其更改为其他端口号.

    因为在组策略设置了需要按ctrl+alt+del才能登陆,想取消的话在运行里输入gpedit.msc打开组策略然后打开——计算机设置 ——安全设置 ——本地策略 ——安全选项 ——交互登陆:无须按ctrl+alt+del

    博客可以参考这篇：http://blog.sina.com.cn/s/blog_6d1b3b900102wkb5.html-

  2. 设置安全组策略，不允许SAM账户的匿名枚举，不允许SAM账户和共享的匿名枚举。

  3. 在组策略中设置禁止访问注册表编辑工具

    win+R: gpedit.msc的字符，进入组策略的界面

    在组策略的界面中双击”用户配置”的管理模板，”阻止访问注册表编辑工具”

  4. 开启审核策略

  5. 禁止445端口漏洞

    选择网络连接的本地连接。在本地连接的界面中，把”microsoft”网络中的文件和打印机共享”的单选框去掉

  6. 设置屏幕保护在恢复时使用密码保护

  7. 设置windows密码策略，使密码最小长度为8为，最长预留期为30天。锁定时间为30分钟，设置账户锁定阙值为6次无效登录

  8. 开启Windows防火墙，关闭ping服务
