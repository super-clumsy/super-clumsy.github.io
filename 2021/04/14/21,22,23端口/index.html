之前只知道==telnet(23端口)==是一个危险的端口，一般不开，但是，甚至不知道他的用法什么的，，这会乘着摸鱼的功夫总结一下

telent为什么危险

1、Telnet当成一种通信协议，但是对于入侵者而言，Telnet只是一种远程登录的工具。一旦用户与远程主机建立了Telnet连接，用户便可以使用目标主机上的软、硬件资源，而用户的本地机只相当于一个只有键盘和显示器的终端而已。

2、telnet命令用于登录远程主机，对远程主机进行管理。telnet因为采用明文传送报文，安全性不好，很多Linux服务器都不开放telnet服务，而改用更安全的ssh方式了。

Telnet 命令为用户提供了在本地计算机上完成远程主机工作的能力，它可以使用户坐在已上网的电脑键盘前通过网络进入的另一台电脑已上网的电脑，使它们互相连通。

在终端使用者的电脑上使用telnet程序，用它连接到服务器。终端使用者可以在telnet程序中输入命令，这些命令会在服务器上运行，就像直接在服务器的控制台上输入一样。

centos7演示telnet

安装

检测是否安装

rpm -qa | grep telnet

rpm -qa | grep xinetd

rpm命令使用

rpm -qa 查询所有安装包的信息

rpm -qi 查询包的详细信息

rpm -ql 查询包的文件列表

rpm -qf 查询系统文件属于哪个rpm包

rpm -qR 查询包的依赖关系

rpm -e 删除包名

如果没有安装，首先应该安装

安装

yum -y install telnet-server.x86_64

yum -y install telnet.x86_64

yum -y install xinetd.x86_64

注意：elnet-server服务启动依赖xinetd服务

xinetd：eXtended InterNET services daemon，超级Internet服务器，用来管理多种轻量级Internet服务。

FTP服务的搭建

先确认是否有安装包

rpm -q vsftpd

如果没有的话，自然要安装

yum -y install vsftpd

启动FTP服务

systemctl start vsftpd.service

之后可以设置一些权限，和用户名什么的，也可以确定是否可以匿名登录之类的。

ssh

ssh 是22端口，大部分适用于生成公钥私钥，用作免密登录。

openssl

openSSL是SSL/TLS协议的开源实现，始于1998年，支持Windows，mac，linux平台

常用命令

生成私钥：openssl genrsa -out my,key

生成公钥：openssl rsa -in my.key -pubout -out my.pem

可以使用OpenSSL构建一套属于自己的CA，给自己颁发证书，称为”自证书签名”
