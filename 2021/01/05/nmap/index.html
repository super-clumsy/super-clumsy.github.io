nmap的基本使用

Nmap是一款开源的网络探测和安全审核工具。它被设计用来快速扫描大型网络，包括主机探测与发现、开放的端口情况、操作系统与应用服务指纹识别、WAF识别以及常见的安全漏洞。

Nmap的使用参数非常之多，总结的是一些常用到参数。

大致从网上可以了解到的思维导图有

基本使用

1.描目标地址(ip)

nmap 192.168.xx

2.扫描域名

nmap goole.com

返回结果

可以见到，开放了，80，81，443端口。

以上两种扫描都是直接扫描，直接扫描速度快，可以作为一个简单的收集前扫描

3.扫描目标地址所在的网段

nmap 192.168.254.0/24

24呢，这里指的是，一个网段 (0-255), 二进制的24个1

4.指定端口扫描

nmap 192.168.xxx -p 21,22,23,80,3389

5.扫描目标地址c段的在线主机状况

-sP : nmap 仅对主机进行ping扫描（并不进行端口探测，还有操作系统识别），这个命令仅仅是列出了这个主机是否存活，或者这一个网段内都有哪些主机是在存活状态。

nmap -sP 192.168.254.0/24

对这个C段进行网络扫描

6.对目标地址进行路由追踪

nmap --traceroute 192.168.254.129

7.识别目标系统指纹

namp -O 192.168.254.129

8.目标开放端口版本检测

nmap -sV 192.168.254.129

9.探测目标防火墙的状态

nmap -sF -T4 192.168.254.129 这是使用扫描级别为T4，一般在网络通信良好的情况下使用T4

-T0 偏执的：非常非常慢，用于IDS逃逸

-T1 猥琐的：相当慢，用于IDS逃逸

-T2 有礼貌的：降低速度以消耗更小的带宽，比默认慢十倍

-T3 普通的：默认，根据目标的反应自动调整时间模式

-T4 野蛮的：假定处在一个很好的网络环境，请求可能会淹没目标

-T5 疯狂的：非常野蛮，很可能会淹没目标端口或是漏掉一些开放端口

10.扫描文件内容的ip

nmap -iL ip.txt

11.随机选取ip

nmap -iR 3 -v(-v输出详细信息)

12.exclude(排除)

nmap -iL ip.txt exclude baidu.com (排除baidu.com，扫描)

13.excludefile

排除目标文件

nmap 的脚本使用

鉴权扫描(弱口令检测)

nmap --script=auth 192.168.254.129

暴力破解

nmap --script=brute 192.168.254.129

应用服务扫描

nmap --script=realvnc-auth-bypass 192.168.254.129

局域网服务探测

nmap -n -p 445 --script=broadcast 192.168.254.129 - n参数是不进行DNS解析

whois查询

nmap -script external baidu.com

常见参数

-sn:ping扫描，只进行主机发现，不进行端口扫描。在新的nmap中sn代替了sP
-Pn:穿透防火墙，一般在扫描时都会添加
-F:快速扫描
-v:显示扫描过程
-sS:TCP SYN扫描
-A:全面系统扫描
-sU:UDP扫描
-sT:TCP扫描
-n/-R:不进行DNS解析/总是进行DNS解析
–script=vuln:漏洞扫描
–script=dns-brute +域名:DNS扫描
–script=whois-domain +域名 :whois扫描

端口扫描状态

Opend:端口打开
Closed：端口关闭
Filtered：端口被过滤，数据没有到达主机，返回结果为空，数据被防火墙
Unfiltered：未被过滤，数据有到达主机，但是不能识别端口当前状态
Open|filtered:开放或者被过滤，端口没有返回值，主要发生在UDP、IP、FIN、NULL和Xmas扫描中
Closed|filtered:开放或者被过滤，只发生在IP ID idle扫描
