一些零散的知识点的笔记

jsonp

首先知道

  1. jsonp的核心原理就是目标页面回调本地页面的方法,并带入参数
  2. 我们常用的动态页面有jsp,php,aspx

为什么需要jsonp

  1. 由于浏览器安全限制,数据是不可以直接跨域(包括不同的根域名、二级域名、或不同的端口)请求的,除非目标域名授权你可以访问。比如设置crossdomain.xml 或在http头部里授权

  2. 但是crossdomain.xml会允许设置的网站访问所有的数据,而头部设置又非常麻烦。

  3. 所以可以在你授权的数据返回里设置jsonp来让该接口允许所有的调用者获取数据。

jsonp的原理

jsonp是使用方法回调的原理.

在网页里,你如果引入其他网页的 js, 那这个页面的js是可以调用你网页的代码的

直接请求js 和 请求的动态页面(jsp,php,aspx)里输出的javascript代码 效果一样

如果引用的js或动态页面里有 showjson({“url”:”http://www.bejson.com“});这行代码的话,那就会弹出 http://www.bejson.com

SQL与ORM的优缺点

相对来说，ORM的缺点就是SQL的优势地方，而优点也是SQL的劣势地方。

  • 优点

    • 方便的使用面向对象，语句清晰
    • 防注入『这个其实不算ORM的核心，因为比如Phalcon的SQL形式写法也可以防注入』
    • 方便动态构造语句，对于不同的表的相同操作采用多态实现更优雅
    • 一定程度方便重构数据层『比如改表名，字段名等』
    • 设置钩子函数

  • 缺点

    • 不太容易处理复杂查询语句
    • 性能较直接用SQL差

    如何选择

    尽量使用ORM，除了含子查询的复杂语句『不过大流量下的网站最好不要写这种复杂SQL』，当然这也只是一个原则，所以反过来说也可以，不过以下几个场景用ORM的好处是很明显的

    • Model对象不确定的时候，使用多态的方式去处理不同实例的相同操作
    • 语句结构不确定的时候，比如根据不同的情况Where子句不一样的时候，采用if的代码结构去控制ORM方法的使用比拼接SQL语句要清晰的多
    • 设置钩子函数，比如分页里面，拿分页数据同时要count数据，那么就可以在Model里面插入这个算count的钩子函数（包括缓存逻辑）

JNDI

JNDI是==Java Naming and Directory Interface==，是SUN公司提供的一种，标准的JAVA命名系统接口，JNDI提供统一的客户端的API，通过不同的访问提供接口者接口JNDI服务供应接口实现，由管理者将JDNI API映射为特定的命名服务和目录 系统，使得java应用程序可以和这些命名的服务和目录之间进行交互。目录服务是命名服务的一种自然扩展，两者之间的关键差别是目录服务中，不但有对象，还可以有属性和名称。而命名服务对象中没有属性。

集群JNDI实现了高可靠性JNDI，通过服务器的集群，保证了JNDI的负载平衡和错误恢复。在全局共享的方式下，集群中的一个应用服务器保证本地JNDI树的独立性，并拥有全局的JNDI树。每个应用服务器在把部署的服务对象绑定到自己本地的JNDI树的同时，还绑定到一个共享的全局JNDI树，实现全局JNDI和自身JNDI的联系。

DNS、XNam 、Novell目录服务、LDAP(Lightweight Directory Access Protocol轻型目录访问协议)、 CORBA对象服务、文件系统、Windows XP/2000/NT/Me/9x的注册表、RMI、DSML v1&v2、NIS。

LDAP

LDAP是轻量目录访问协议，英文全称是Lightweight Directory Access Protocol，一般都简称为LDAP。它是基于X.500标准的，但是简单得多并且可以根据需要定制。

  • LDAP是一种通讯协议，LDAP支持TCP/IP。协议就是标准，并且是抽象的，在这套标准下，AD(Active Directory)，是微软出的一套实现。
  • AD呢，咱们可以把他理解为数据库。也有很多人直接把LDAP直接说成数据库(可以把LADP理解成储存数据的数据库)。像是其他数据库一样。LADP也是有client端和server端。server端用来存放资源，client端用来增删改查等操作。
  • 而我们通常说的LDAP是指运行这个数据库的服务器。
  • 可以简单理解AD=LADP服务器+LADP应用

那么，LADP这种数据库有什么特殊的呢？

  • Mysql数据库，数据都是按记录一条条记录存在表中。而LADP数据库，是数据结构上的，数据储存在叶子节点上的。

什么是LDAP

什么事目录服务

  1. 目录服务是一个特殊的数据库，用来保存描述性的，基于属性的详细信息。基于属性的详细功能。支持过滤的功能
  2. 是动态的，灵活的，易于扩展的。

如：人员组织管理，电话簿，地址簿

LDAP介绍

目录是一个为查询，浏览和搜索而优化的数据库，它成树状结构组织数据。类似文件目录一样。

目录数据库和关系数据库不同，他有优异的读性能，但写性能差，并且没有事务处理，回滚等复杂的功能，不适于储存修改频繁的数据。所以目录天生是用来查询的，就像他的名字一样。

LDAP目录服务是由于目录数据库和一套访问协议组成的系统。

为什么要用

LDAP是开放的internet标准，支持跨平台的internet协议，在业界种得到广泛认可，并且市场上或者开源社区上的产品都加入了对LDAP的支持，因此对于这类系统，不需要单独定制，只需要LDAP做简单的配置，就可以与服务器做交互认证。

基本概念

  1. 目录树：在一个目录服务系统中，整个目录信息集可以表示为一个目录信息树，树中的每个节点是一个条目。

  2. 条目：每个条目就是一条记录，每个条目有自己的唯一可区别的名称（DN）。

  3. 对象类：与某个实体类型对应的一组属性，对象类是可以继承的，这样父类的必须属性也会被继承下来。

  4. 属性：描述条目的某个方面的信息，一个属性由一个属性类型和一个或多个属性值组成，属性有必须属性和非必须属性。
